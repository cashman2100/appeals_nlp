# appeals_nlp

**Иерархическая классификация русскоязычных обращений ЖКХ под шумом ASR.**

Проект для курса *Natural Language Processing & LLMs*. По тексту обращения
жителя (транскрипт телефонного звонка голосовому роботу) предсказывается
категория заявки. В качестве честного baseline используется production
rule-based система. Отдельно исследуется задача восстановления категории для
обращений, которые автоматическая система не смогла классифицировать.

## Данные

98 189 обращений, 5 управляющих компаний, период 2020–2026, только обращения
через голосового робота. Подробное описание — [`data/DATACARD.md`](data/DATACARD.md).
Датасет содержит корпоративные данные и предоставляется приватно для верификации;
весь код открыт.

## Структура

```
configs/        конфигурация и таксономия категорий
src/data/       препроцессинг, EDA, анонимизация
src/models/     baseline-модели, нейросетевой классификатор
src/training/   обучение
src/evaluation/ метрики, анализ ошибок
notebooks/      воспроизводимые ноутбуки
results/        журнал экспериментов

```

## Воспроизведение

```bash
pip install -r requirements.txt
python -m src.data.preprocess --config configs/base.yaml
python -m src.data.eda        --config configs/base.yaml
```

## Постановка задачи

Вход: текст обращения (ASR-транскрипт, разговорный русский, шум распознавания).
Выход: категория заявки (топ-14 + «Прочее»). Метрика — macro-F1 (сильный
дисбаланс классов). Сплит — временной (обучение 2020–2024, тест 2025–2026),
что имитирует реальное применение и учитывает дрейф распределения.

## Подходы

Сравнивается несколько подходов нарастающей сложности: production rule-based
baseline, классические методы (TF-IDF + линейные модели, fastText) и дообучение
трансформерных энкодеров для русского языка.
