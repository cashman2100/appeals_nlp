# Dataset Card: Appeals-NLP

## Краткое описание

**Appeals-NLP** — корпус русскоязычных обращений жителей в диспетчерскую службу ЖКХ,
полученных через голосового робота (ASR-транскрипция телефонных звонков).
Каждое обращение размечено категорией, приоритетом и (частично) службой исполнителя
силами реальной production-системы и диспетчеров.

| Параметр | Значение |
|---|---|
| Размер | 98 189 обращений |
| Уникальных описаний | 79 556 (81%) |
| Управляющих компаний | 5 |
| Период | 2020–2026 (7 лет) |
| Язык | Русский (разговорный, ASR-шум) |
| Источник | Только голосовой робот (`creator = 'robot'`) |
| Медианная длина | 44 символа |

## Происхождение данных

Житель звонит в диспетчерскую, голосовой робот задаёт вопросы (в т.ч. «что случилось?»),
распознаёт речь и формирует заявку. Робот по таблице слов-синонимов выбирает категорию;
по категории production-скрипт назначает приоритет и службу. Диспетчеры могут править
категорию/приоритет/исполнителя вручную — поэтому в данных представлены **итоговые**
(post-edit) метки, а не исходный выбор робота (исходные в БД не логируются).

## Поля

| Поле | Тип | Роль |
|---|---|---|
| `customer_id` | int | ID управляющей компании (5 значений) |
| `year` | int | Год создания (для temporal split) |
| `month_name` | str | Месяц (сезонность; «Отопление» концентрируется окт–янв) |
| `description` | str | **Главный вход.** ASR-транскрипт обращения |
| `category_parent` | str | Родительская категория (Авария / Текущая / …); NaN для «Не определена» |
| `category_child` | str | Дочерняя категория (55 значений, сырая — сохраняется всегда) |
| `priority` | str | Срочно / Высокий / Средний / Низкий |
| `performer_position` | str | Должность исполнителя (нормализуется в 20 групп; ~34% NaN) |
| `work_label` | str | **Целевая метка** (производная): топ-14 + «Прочее»; «Не определена» → `UNDEFINED` |

Таксономия рабочих меток задаётся `configs/category_mapping_v1.json`. Смена таксономии
(все 55 классов / иерархия parent→child) = смена mapping-файла, **без переразметки данных**.

## Сплиты (temporal)

Имитируем production: учимся на прошлом, тестируем на будущем.

| Сплит | Годы | Размер | Примечание |
|---|---|---|---|
| train | 2020–2024 | 46 030 | дубли сохранены (реальная частотность — сигнал) |
| val | хвост train | 5 626 | последние 10% train по времени, уникальные |
| test | 2025–2026 | 30 510 | только уникальные описания, anti-leak с train |
| UNDEFINED pool | все | 9 955 | для трека реклассификации (не входит в train/test category) |

Наблюдается **реальный temporal drift** долей классов (напр. «Водоснабжение» train 23% →
test 17%) — подтверждает, что temporal split честнее случайного.

## Особенности и шум

- **ASR-стиль**: 74% текстов целиком в нижнем регистре, лишь 16% содержат пунктуацию.
- **Короткие**: 22% описаний < 25 символов («холодные батареи», «нет света»).
- **«Не определена» (10%)**: робот не смог классифицировать → ушло диспетчеру. Все
  с непустым текстом → ценнейший таргет для задачи реклассификации.
- **Дубли (19%)**: повторяющиеся реальные заявки («нет горячей воды» × сотни). В train
  сохраняем (отражает prior), в val/test дедуплицируем (честная оценка, без утечки).
- **Приоритет**: не строго детерминирован категорией (средняя доля доминирующего
  приоритета внутри (customer, parent, child) ≈ 85%, не 100%) → задача осмысленна.

## Этика и приватность

Корпоративные данные. В описаниях возможны ПДн (адреса, имена, телефоны). Для публичного
релиза предусмотрен анонимизирующий pipeline (regex + NER маскирование) — см. Limitations
в отчёте. Текущая версия — приватная, доступ для проверки проекта по запросу.
